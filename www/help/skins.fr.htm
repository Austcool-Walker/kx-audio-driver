Sélecteur d'apparence (skins)

Différentes apparences peuvent être utilisées par le mixeur kX.

L'architecture de gestion des apparences a été étudiée en vue de faciliter la création de nouvelles apparences pour le mixeur kX. La procédure pour créer de nouvelles apparences est la suivante :

  • Si vous n'avez pas l'intention d'ajouter ou de déplacer certains contrôles, alors vous pouvez simplement créer votre fichier .zip avec des images .jpg ou .gif, puis créer un fichier kxskin.ini personnalisé avec seulement une section, appelée '[skin]', qui décrira votre nouvelle apparence. N'oubliez pas d'ajouter le fichier kxskin.ini à votre archive zip. Votre nouveau fichier zip peut être situé n'importe où sur votre disque dur. Son nom ne doit pas être kxskin.zip et vous devez remplacer la chaîne 'GUID' par une nouvelle, générée par le mixeur (Configuration ->Générer un Guid). (La signification de GUID est IDentifieur Globalement Unique/Globally Unique IDentifier).
  • Si votre apparence nécessite de changer la taille des fenêtres ou leur emplacement, vous devez créer un fichier kxskin.ini plus sophistiqué. Pour plus de détails regarder l'exemple de kxskin.ini inclus. Cependant, si certains paramétrages sont omis dans votre apparence, ils seront automatiquement remplacés par les valeurs par défaut du kxskin.ini intégré avec le mixeur kX. De cette manière, vous n'êtes tenu que de fournir les parties spécifiques à votre apparence dans votre fichier kxskin.ini (avec au minimum la section [skin]'). Les images (.jpg/.gif files) et fichiers son (.wav) sont dorénavant automatiquement récupérés à partir du fichier kxskin.zip quand vous utilisez un mixeur personnalisé (depuis la version 3514).
  • D'une manière générale, les noms de fichiers image et son référencés par le kxskin.ini ne doivent pas être modifiés, vous de ne devez pas, de la même manière, modifier la chaîne de texte (en fait, vous pouvez les omettre dans votre kxskin.ini personnalisé). Cependant, si vous envisagez de faire une version dans un langage particulier prenez contact avec nous par courrier électronique.
  • Une fois vos images réalisées, celles ci doivent être compressées (format ZIP) avec votre kxskin.ini personnalisé vers un fichier unique ainsi qu'avec un GUID unique. Pour utiliser la nouvelle apparence, cliquer sur 'Sélecteur d'apparence -> Nouvelle apparence'.
  • Notez également que le nom de l'apparence doit également être unique.
  • Pour utiliser les modules d'apparence vous devez avoir au minimum la version 3509 du logiciel kX.
  • Depuis la version 3521 vous pouvez utiliser des archives de type RAR à la place du type ZIP. Cependant, éviter les archives de type 'solides' qui sont susceptibles de réduite les performances globales.
  • Depuis la version 3523 les informations concernant les paramètres régionaux sont stockées dans le fichier lang.ini. N'ajouter pas ce fichier dans vos apparences personnalisées.
  • Rappelez vous que tous les fichiers non trouvés dans votre fichier personnalisé sont extraits du fichier par défaut. Par exemple vous pouvez tout à fait ne pas inclure de fichiers .wav dans votre apparence personnalisée.
